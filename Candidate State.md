
# Algorithmic Principle: Candidate State 

**Definition**

A candidate state is a temporary solution maintained until sufficient evidence exists to commit it as the current best solution. 


**Purpose**

Separate exploration from commitment. 


**Invariant**

A candidate may change frequently. 
The committed answer changes only when the candidate improves the objective. 


**Examples**

1. [[Kadane's Algorithm]]
2. [[Boyer-Moore Majority Vote Algorithm]]
3. Greedy Algorithms
4. Graph Algorithms

- BFS frontier
- DP states
- Sliding Window 
- Constraint Filtering
